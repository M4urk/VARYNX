# Contributing to VARYNX

Thank you for your interest in contributing to VARYNX!  
This project is focused on building a privacy‑first, defensive‑only mobile guardian.

## How to Contribute

### 1. Reporting Issues
If you find a bug or have a feature request:
- Open an issue on GitHub
- Include steps to reproduce (if applicable)
- Include screenshots or logs when helpful

### 2. Submitting Pull Requests
- Fork the repository
- Create a new branch for your change
- Follow existing code style and structure
- Submit a pull request with a clear description

### 3. Code Style
**Android (Kotlin):**
- Use Jetpack standards
- Keep Activities/Fragments clean
- Prefer ViewModels where appropriate

**Python:**
- Follow PEP8
- Keep modules small and focused
- Use constructor injection for engine wiring

### 4. Security Contributions
If your contribution relates to security, please read `SECURITY.md`.

## Code of Conduct
By participating, you agree to follow the project’s `CODE_OF_CONDUCT.md`.
